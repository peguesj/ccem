VIKI — Vectorized Intelligence & Knowledge Infrastructure

VIKI is a self-hosted AI conversation memory and retrieval platform. It captures, compresses, and semantically indexes all AI conversations and screen recordings across multiple AI platforms, creating a personal data lake with an enterprise-grade RAG engine.

Primary System
Multi-format conversation storage (Parquet + JSON). Vector search with Azure AI Search. Medallion architecture (Bronze/Silver/Gold). Event-driven ingestion pipeline.
Companion: OSYJ
Open Source Your Journey. Personal development tracking. AI coaching (GPT-4o). Strategic thinking frameworks. Life event analytics. Multi-modal capture.
Current Status
Active Development · feature/vikiagent-resource-dashboard. CP-50 in progress. 10,517 chunks captured. 9 Azure services in East US production.

Architecture Overview

4-Layer System Architecture
Medallion Architecture
BronzeRaw immutable data (Parquet + Zstd). Partitioned by date/platform/user. 1TB capacity, 90-day retention.
SilverValidated & normalized data. Schema enforcement. Deduplication. Quality checks.
GoldAnalytics-ready aggregations. Daily/weekly/monthly rollups. Pre-computed metrics. Business intelligence.

Public API Endpoints

No authentication required. These endpoints are accessible without a bearer token.

GET /api/health HealthController

Returns system health status. Used by VikiAgent polling and external monitoring.

{"status": "ok", "version": "0.1.0", "timestamp": "2026-03-16T..."}
GET /api/sync/status SyncStatusController

Returns current formation sync status. Polled by Python formation agents to check system state.

{"status": "running", "agents": 23, "chunks_processed": 10517, "last_checkin": "..."}
POST /api/sync/status SyncStatusController#checkin

Formation agent check-in. Updates last-seen timestamp and progress data.

Body: {"agent_id": "agent-01", "chunks": 450, "status": "running"}
GET /api/insights/pending InsightsController

Returns pending AI insights for delivery to VikiAgent. Polled every 5 minutes by VikiAgent.

POST /api/insights/:id/acknowledge InsightsController

Mark an insight as delivered. Called by VikiAgent after displaying OS notification.

GET /api/expertise/search InsightsController

Full-text search over viki_knowledge_items. Query param: ?q=elixir

GET /api/viki-interface/settings VikiInterfaceController

Returns VIKI Interface display settings (tile layout, face blend mode, speech config).

WebAuthn / Passkey API

Requires active browser session (cookie). Implements W3C WebAuthn Level 2 via wax_ 0.6 library. Passkeys stored in viki_webauthn_credentials table.

MethodEndpointDescription
GET/api/passkey/register/optionsGet WebAuthn registration challenge
POST/api/passkey/registerComplete passkey registration with attestation
GET/api/passkey/authenticate/optionsGet WebAuthn authentication challenge
POST/api/passkey/authenticateComplete passkey authentication + OTP redirect if return_url present
VikiAgent Auth Flow
1. VikiAgent: "Authenticate" menu action
2. Opens system browser: /login?return_url=vikiagent://auth/callback
3. User authenticates via passkey (Touch ID / Face ID)
4. PasskeyController generates OTP → redirects to /auth/redeem?token=OTP&return_url=vikiagent://...
5. AuthTokenController validates OTP → redirects to vikiagent://auth/callback?session=...
6. VikiAgent handles deep link → extracts session token
7. WKWebView loads /dashboard with session token

Formation — Overview

The "formation" is a parallel Python agent architecture for batch-processing 22 months of historical MemoryVault content. 23 agents run concurrently, each responsible for a time slice of recordings.

Architecture
• 23 parallel Python processes
• Each agent handles a date range slice
• ffmpeg + libsvtav1 for AV1 compression
• Opus audio at 96kbps
• PID file: /tmp/mv-logs/pids.txt
• Log dir: /tmp/mv-logs/agent-NN.log
• Status: POST /api/sync/status

Security

Auth & Identity
• Azure AD — primary identity provider
• Managed Identities — service-to-service
• WebAuthn/Passkey — user auth (wax_ 0.6)

Search indexer failures
curl "$SEARCH_ENDPOINT/indexers/viki-conversations-indexer/status?api-version=2023-11-01" -H "api-key: $SEARCH_KEY"
# Reset and rerun:
curl -X POST "$SEARCH_ENDPOINT/indexers/viki-conversations-indexer/reset?api-version=2023-11-01" -H "api-key: $SEARCH_KEY"
Formation agent not starting
ls /tmp/mv-logs/        # check log directory exists
cat /tmp/mv-logs/pids.txt    # check PIDs file
tail -50 /tmp/mv-logs/agent-01.log   # check agent logs
ps aux | grep backfill   # check processes are running
Phoenix won't start (port conflict)
lsof -ti:6769 | xargs kill -9   # kill process on port 6769
mix phx.server                   # restart Phoenix
Enable debug logging
export LOG_LEVEL=DEBUG
export AZURE_LOG_LEVEL=DEBUG
mix phx.server
