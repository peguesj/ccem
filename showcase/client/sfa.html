Release Notes — v4.7.0-dev

Universal MIDI Transport

All LiveViews now subscribe to the midi:actions PubSub topic. CrateDigger and SampleLibrary handle :play/:stop/:next_track/:prev_track uniformly. Physical controller actions propagate across the entire app with zero configuration.

ADR-004 UX Overhaul (DJ Industry Standards)

Hot cue pads redesigned to 2×4 grid (Rekordbox/Traktor standard) with color-coded letters (A=red, B=blue, C=green…). Empty pads show dimmed base color as affordance. CrateDigger track rows now display BPM and key inline. SampleLibrary play button always visible (opacity-40) per Serato pattern.

Comprehensive E2E Test Coverage

Three Playwright spec files: full-pipeline.spec.ts (9 steps), dj-daw-midi.spec.ts (15 cases), crate-sample-library.spec.ts (12 cases). Tests verify AppHeader on all routes, hot cue grid layout, play button opacity, crossfader curve inline placement.

SampleLibrary LiveView (New Route)

Full-featured sample browser at /samples. Search + BPM range + key + category filters. SamplePacks sidebar. Streaming file table with preview via SamplePreview JS hook. AppHeader tab added. MIDI transport controls wired.

Docs Site + Full Changelog

CHANGELOG.md updated with every version from v1.0.0 through v4.7.0-dev. Version jump dropdown added to Jekyll topbar and inside the changelog page. All 12 v4.2.x patch versions documented individually.
