# CCEM - Claude Code Environment Manager

## Overview
Comprehensive TUI-based configuration management system for Claude Code v2.0.10 with fork point discovery, chat history analysis, and schema-based architecture.

## Strategic Goals
1. **TDD-First Development**: 95%+ test coverage with Jest + TypeScript + ink-testing-library
2. **TUI Excellence**: Rich terminal interface with Ink + React
3. **Schema-First Design**: Zod validation with UUID-based relationships
4. **Fork Discovery**: Chat history analysis and git worktree detection
5. **Documentation**: TSDoc standard with automated generation
6. **Semantic Versioning**: TypeScript-specific SemVer compliance

## Technical Stack
- **Language**: TypeScript 5.3+ (strict mode)
- **Runtime**: Node.js 18+
- **TUI**: Ink 4+ with React 18
- **Testing**: Jest 29+ with ts-jest + ink-testing-library
- **Validation**: Zod 3.22+
- **Documentation**: TSDoc + TypeDoc
- **Build**: esbuild/tsup

## Implementation Methodology
**Test-Driven Development** (Uncle Bob's Three Rules):
1. Write failing test first (RED)
2. Write minimum code to pass (GREEN)
3. Refactor while maintaining tests (REFACTOR)
